Comandos utilizados:

git clone: Se utiliza para clonar un repositorio Git existente en tu máquina local. Copia todos los archivos y la historia del repositorio remoto en tu directorio de trabajo local.
            Lo he usado para clonar el repositorio: https://github.com/gitt-3-pat/p1 en mi fork.
            Cloning into 'p1'...
            remote: Enumerating objects: 6, done.
            remote: Counting objects: 100% (6/6), done.
            remote: Compressing objects: 100% (3/3), done.
            remote: Total 6 (delta 0), reused 1 (delta 0), pack-reused 0
            Receiving objects: 100% (6/6), done.

git status: Muestra el estado actual de tu directorio de trabajo local en relación con el repositorio Git. Puedes ver qué archivos han sido modificados, añadidos o eliminados desde el último commit.
            On branch main
            Your branch is up to date with 'origin/main'.

            Changes not staged for commit:
            (use "git add <file>..." to update what will be committed)
            (use "git restore <file>..." to discard changes in working directory)
                    modified:   git.txt
            Aparece esto porque estoy modificando git.txt después de haberlo creado.

git add: Se utiliza para agregar cambios al área de preparación antes de realizar un commit. Puedes agregar archivos específicos o todos los archivos modificados.
        Lo he utilizado para agregar git.txt y entorno.pdf. No aparece nada en la terminal, simplemente lo agrega.

git commit: Captura un snapshot de los cambios que has realizado en el área de preparación y los guarda en el historial del repositorio. Debes incluir un mensaje descriptivo que explique los cambios realizados.
            [main 114b2f7] feat: homepage
            1 file changed, 0 insertions(+), 0 deletions(-)
            create mode 100644 entorno.pdf
            Esto apareció cuando hice commit al crear entorno.pdf, lo que me indica que se ha guardado correctamente.

git push: Se utiliza para enviar tus commits locales al repositorio remoto. Específicamente, actualiza la rama remota con tus cambios.
        Enumerating objects: 4, done.
        Counting objects: 100% (4/4), done.
        Delta compression using up to 2 threads
        Compressing objects: 100% (2/2), done.
        Writing objects: 100% (3/3), 353 bytes | 353.00 KiB/s, done.
        Total 3 (delta 0), reused 0 (delta 0), pack-reused 0
        To https://github.com/paulaap16/p1-fork
        059f902..114b2f7  main -> main

git checkout: Puede tener diferentes funciones dependiendo de cómo se utilice, pero comúnmente se usa para cambiar entre ramas o para restaurar archivos a un estado específico.
